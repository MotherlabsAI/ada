# UNK.103 · True first-run onboarding — is recognition enough?

> Ω residue · L1 · C0 · area **UNK** · from `SEED (root intent)`

## Summary
The welcome assumes the user knows what a 'pack', a 'node', a 'graph' is. For a genuine first-ever run (never seen Ada, non-technical) recognition-over-recall may not be enough — there may need to be a one-time guided compile. Unverified whether the menu alone carries a true novice.

## Why it matters
First-run is where non-technical users are won or lost; the current design is optimised for the returning 'Alex', not the stranger.

## Failure if missing
A first-time non-technical user recognises the buttons but not the concepts, compiles nothing, and leaves.
