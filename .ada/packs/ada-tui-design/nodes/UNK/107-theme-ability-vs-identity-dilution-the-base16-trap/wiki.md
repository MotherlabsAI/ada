# UNK.107 · Theme-ability vs identity dilution — the base16 trap

> Ω residue · L1 · C0 · area **UNK** · from `src/tui/ink/tokens.ts`

## Summary
tokens.ts is built to re-skin the whole shell from one file — inviting user themes. But the more a user remaps, the less Ada looks like Ada (the base16/ngrok identity trap the palette work warns about). How much customization the surface should expose before the brand dissolves is an open product line.

## Why it matters
It's a direct tension between the token system's flexibility and the 'sister to Claude Code' identity that flexibility could erase.

## Failure if missing
Full theme exposure turns every install into a different-looking tool; the recognisable Ada identity evaporates.
