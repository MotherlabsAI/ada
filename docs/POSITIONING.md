# Positioning — the one-line that defines Ada

> **Ada is to language what Claude Code is to code.**

Claude Code made _code_ a material you can engineer: read it, navigate it, verify it,
compile intent into it. Code had compilers, types, and tests for decades. **Meaning
never did** — it stayed vibes. Ada points the same machine at meaning.

## The structural mapping

| Claude Code → **code**                                       | Ada → **language / meaning**                                                       |
| ------------------------------------------------------------ | ---------------------------------------------------------------------------------- |
| input: NL intent about software                              | input: NL intent about anything                                                    |
| loop: gather context → act with tools → **verify** → repeat  | loop: gather context → **structure into capsules/graph** → **verify (C)** → repeat |
| harness manages: context window, tools, permissions, sandbox | harness manages: context streams, a workforce, quality gates                       |
| the model reasons; you shape the environment                 | the workforce excavates; you shape and **judge** the meaning                       |
| output: working software                                     | output: a working **world model** — graph, wiki, capsules, unknowns                |
| verification: tests, types, build, lint (deterministic)      | verification: **C** where meaning is checkable; **human gate** where it is not     |
| executor: runs the code                                      | executor: Claude Code, which runs the code **from Ada's context**                  |

## What this locks

**C is tests-for-meaning.** Software is mostly human judgment with a deterministic
sliver (tests/types). Meaning is the same ratio: mostly the operator's taste gate, with a
deterministic sliver (C). So C is _supporting trust infrastructure_, not the product —
exactly as a test suite is not the product. (This is why testing "does C catch code bugs"
was a category error: Ada's substrate is meaning, not code.)

**Ada is Claude Code's sibling, one layer up — not a competitor.** It compiles meaning and
hands the result to the thing that compiles code. Ada → context → Claude Code → software.
The CLI should feel like Claude Code's sister (same loop, different face), because it _is_
the same idea applied to the layer above.

**The product is the compile of meaning.** The graph is the topology, the wiki is the
readable memory, the blueprint is the deterministic build contract, C is the trust engine,
Claude Code is the executor. The bet is the _quality of the extracted semantics_ — and the
non-negotiable is that the first node a user opens must impress, because that is the
visible proof that meaning was engineered, not generated.

## The bet, at its root

Code became engineerable when it got compilers + tests. Meaning hasn't had that.
Ada gives meaning the same affordances — structure (graph), navigation (TUI), memory
(wiki), verification (C), and a compile step — so a builder can engineer the _context_
that has to be right before code is even the question.
