# Ada — Growth Strategy

**Authority:** defines how Ada spreads and who it spreads to.
Derived from: BRAND.md, CAPABILITIES.md, VIDEO_DEMO.md.
All growth assumptions are grounded in verifiable product behavior, not marketing goals.

---

## The Growth Equation

Ada spreads when someone uses it, gets a CLAUDE.md that matches their intent,
and Claude Code builds correctly from it the first time.

That experience is the artifact. Sharing the artifact is the growth mechanism.
No referral program needed. The GitHub repo is the proof that travels.

---

## The Primary Distribution Artifact

**Not the website. Not the video. The GitHub repository.**

When Ada compiles a project, the output includes a CLAUDE.md visible in the repo root.
Anyone inspecting that repo can see:

- What Ada produced
- That it derived from plain language, not technical specification
- That Claude Code built from it without per-file instruction

A developer who reads a well-structured CLAUDE.md in someone else's repo
and thinks "how did this get generated?" is one command away from using Ada.

The repo is inspectable. The video demo repository is the first proof artifact.
Every project built with Ada becomes a secondary proof artifact.

## The Cold Start Problem

Ada cannot claim adoption it doesn't have. It does not manufacture social proof.

The cold start strategy:

1. The video demo ships first — creates one inspectable proof artifact with a real GitHub repo
2. The website unlocks the SEO surface — people searching for Claude Code limitations find Ada
3. The CLAUDE.md in the demo repo gets forked — proof travels without ongoing effort
4. Each compilation in the wild produces a new repo with a new CLAUDE.md — organic propagation

There is no shortcut before the video demo ships.

## Pricing and Monetization

Not defined here. Not yet.

The first monetization constraint: Ada must be free to try.
If the first session requires payment, the proof-of-value moment never happens.
Freemium with a meaningful free tier is the only viable structure for
a tool whose value is proven in a single session.

When pricing is defined, it will be grounded in: what does one correct build save?
Not: what does the competition charge?

## What Does Not Work

**Paid acquisition:** Ada cannot be sold to someone who hasn't felt the problem.
Ads to a cold audience produce nothing.

**Partnership integrations:** Anthropic does not partner with tools that depend on Claude Code.
The value proposition is complementary, but the channel is user-to-user, not corporate.

**Press and coverage:** Only works after there is something verifiable to cover.
The video demo is the press kit. It needs to exist before outreach.

**Comparison content:** Do not write "Ada vs MEMORY.md" or "Ada vs Cursor Rules" comparisons.
The retrospective/prospective distinction in BRAND.md is sufficient.
Side-by-side comparisons invite objections from users who use both tools.

## The Viral Invariant

For Ada to spread organically, every piece of output must be shareable as proof.

- CLAUDE.md must be readable by a non-technical person
- The repo must be clean and immediately forkable
- The compilation must finish in one session — long or failed compilations kill sharing
- The first experience must work — one bad session does not get shared

**This means product quality is the growth strategy.**
No amount of distribution fixes a session that produces a CLAUDE.md
that doesn't match the original intent. That's not a marketing problem. It's a product problem.

Growth is downstream of the product being trustworthy enough to show someone else.
